# KDL Frontier API Evaluation Client

This folder contains the example client code for evaluating KDL Frontier through the public API.
The leaderboard side only needs the API endpoint, an issued API key, and these scripts.

Do not commit or redistribute the issued API key. Put it in `.env`
or provide it through the evaluation platform's secret manager.

## API Information

```text
API base: http://leaderboard.koreadeep.com
Chat endpoint: http://leaderboard.koreadeep.com/v1/chat/completions
Model: KDL Frontier
Authentication: Authorization: Bearer <issued-token>
```

The API follows an OpenAI-compatible multimodal chat-completions format.

## Files

- `.env.example`: environment template. Copy it to `.env` and set the issued token.
- `health_check.py`: checks API availability, model metadata, and public limits.
- `health_check.sh`: shell wrapper for `health_check.py`.
- `chat_test.py`: sends one image-question request to the chat-completions API.
- `run_test.sh`: shell wrapper for `chat_test.py`.
- `vlmevalkit/`: VLMEvalKit runner and config template for OCRBench_v2.

## Setup

```bash
cp .env.example .env
vi .env
```

Set the issued token:

```env
DEEPFLOW_API_BASE=http://leaderboard.koreadeep.com
DEEPFLOW_API_KEY=<issued-token>
DEEPFLOW_MODEL="KDL Frontier"
```

## Health Check

```bash
bash health_check.sh
```

Expected endpoints:

- `GET /api/v1/live`
- `GET /v1/models`
- `GET /v1/limits`

## Single Request Test

Set `TEST_IMAGE` and `TEST_QUESTION` in `.env`, then run:

```bash
bash run_test.sh
```

Or run the Python client directly:

```bash
python chat_test.py \
  --image /path/to/test.jpg \
  --question "What is written in the image?"
```

## Minimal curl Example

```bash
curl -X POST "http://leaderboard.koreadeep.com/v1/chat/completions" \
  -H "Authorization: Bearer <issued-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "KDL Frontier",
    "stream": false,
    "messages": [
      {
        "role": "user",
        "content": [
          {"type": "text", "text": "What is written in the image?"},
          {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,<base64-image>"}}
        ]
      }
    ]
  }'
```

## Request Limits

Check the current public limits:

```bash
curl http://leaderboard.koreadeep.com/v1/limits
```

The public API uses queue-based overflow handling. If concurrency exceeds the reported
`max_concurrent_requests_per_token`, requests wait in the server queue instead of failing.
For best throughput with VLMEvalKit, set `NPROC` to the reported
`max_concurrent_requests_per_token` or lower.

## OCRBench_v2 with VLMEvalKit

Use the dedicated runner:

```bash
MAX_SAMPLES=0 MODE=all NPROC=10 bash vlmevalkit/run_vlmevalkit.sh
```

See `vlmevalkit/README.en.md` for the full VLMEvalKit instructions.

## What To Send To The Leaderboard Operator

Send these items:

- This repository folder or the zip generated by `vlmevalkit/package_results.sh`.
- API base: `http://leaderboard.koreadeep.com`
- Model name: `KDL Frontier`
- API key: provide the issued token separately through a secure channel.
- Recommended command: `MAX_SAMPLES=0 MODE=all NPROC=10 bash vlmevalkit/run_vlmevalkit.sh`

The server stores request audit logs internally. The audit log is not required for the
leaderboard operator to run evaluation.
